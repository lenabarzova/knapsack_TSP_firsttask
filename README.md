1. Knapsack - задача о рюкзаке. Я использовала расстояние Хэмминга между бинарными векторами решений.

https://programforyou.ru/poleznoe/kak-zadat-argumenti-komandnoi-stroki-v-visual-studio?ysclid=mmrwhw2c8u976598069

Для обработки файлов из папки я воспользовалась инструкцией с сайта выше и добавила (проект-свойства-отладка) командные аргументы в виде пути до папки и файлов (C:\Users\serg0\source\repos\sheblaev_tasks\x64\Debug\data\ks_4_0)

Теперь в моей программе имя входного файла передаётся через командные аргументы. Это сделано для того, чтобы не изменять код каждый раз при проверке нового теста.

knapsack.cpp - файл с реализацией чтения входных данных, алгоритма решения задачи и вывода результата

knapsack.h - заголовочный файл с описанием структур и функций

knapmain.cpp - точка входа в программу. В этом файле происходит получение имени входного файла из командной строки, чтение данных и вызов алгоритма решения задачи

Прикреплены скрины итога работы программы - вывод терминала для двух файлов из data

knapsack_bnb.h — описаны основные структуры данных для задачи, то есть предмет и результат, а также объявлены функции для чтения входного файла, решения задачи методом ветвей и границ и вывода ответа.

knapsack_bnb.cpp — реализован сам метод ветвей и границ: чтение данных из файла, сортировку предметов по удельной ценности, построение верхней и нижней оценок, рекурсивный перебор подзадач с отсечением неперспективных ветвей и формирование лучшего найденного допустимого решения.

main_bnb.cpp — основной запуск программы для метода ветвей и границ, путь к входному файлу передается через командные аргументы

knapsack_ga.h — структуры данных и объявления функций, которые нужны для решения задачи о рюкзаке с помощью генетического алгоритма.

knapsack_ga.cpp — реализован генетический алгоритм: чтение входных данных, представление решения в виде бинарного вектора, генерацию начальной популяции, оценку особей, операции скрещивания и мутации, отбор лучших решений по приспособленности и вывод лучшего найденного набора предметов.

main_ga.cpp — основной запуск программы для генетического алгоритма, путь к входному файлу передается через командные аргументы

2. Задача о коммивояжере  - TSP

tspmain.cpp - точка входа в программу. В этом файле происходит получение имени входного файла из командной строки, чтение данных и вызов алгоритма решения.

tsp.h - заголовочный файл с описанием структур и функций.

tsp.cpp - файл с реализацией чтения входных данных, вычисления длины маршрута, построения начального решения и локального поиска с помощью 2-opt и 3-opt.

Примечание: в программе реализованы оба локальных преобразования: 2-opt и 3-opt. Для небольших входных данных применяются оба метода. Для более крупных тестов основной этап улучшения выполняется с помощью 2-opt, так как полный перебор преобразований 3-opt требует значительно большего времени вычисления. Файлы также передаются через командные аргументы, скрины работы прикреплены (вывод терминала)
